# PROMPT_RULES.md — Prompt 编写规范

> 本文档规定所有发给 AI 助手的 Prompt 格式和要求。
> 以后所有模块开发 Prompt 必须遵循此规范。

---

## Prompt 固定流程

所有开发类 Prompt 必须包含以下 5 个步骤，不得跳过：

### 第一步：恢复项目上下文
```
请先阅读以下文档：
1. AGENTS.md
2. README.md
3. docs/PROJECT.md
4. docs/MODULES.md
5. docs/ARCHITECTURE.md
6. docs/DEVELOPMENT_STATUS.md
7. docs/AI_MEMORY.md
8. 当前模块涉及的相关文档

阅读完成后输出【项目理解】，包括：
- 当前版本
- 已完成模块
- 当前模块
- 下一模块
- 项目状态
- 本次是否需要数据库修改
- 本次是否需要 API 修改
- 本次是否需要 Migration
- 本次需要更新哪些文档
```

### 第二步：分析需求
- 明确本次开发目标
- 确认归属模块
- 检查依赖是否满足
- 列出禁止事项（不修改其他模块、不修改已完成模块等）

### 第三步：开发
- 按模块开发模板逐项执行
- 同步编写测试
- 同步更新文档

### 第四步：更新文档
- CHANGELOG.md（必须）
- MODULES.md（状态变更时）
- DEVELOPMENT_STATUS.md（必须）
- 其他相关文档（按文档维护要求）

### 第五步：输出总结
```
最后输出：
- 新增文件
- 修改文件
- 每个文件作用
- 新增规则
- 下一模块建议
```

---

## Prompt 编写原则

### 必须包含
- ✅ 明确的开发目标（不含糊）
- ✅ 明确的禁止事项（不做什么）
- ✅ 明确的完成标准（什么叫做完）
- ✅ 明确的文档更新要求

### 必须避免
- ❌ 模糊的需求描述
- ❌ 缺少禁止事项（AI 容易过度开发）
- ❌ 缺少文档更新要求
- ❌ 多项任务混在一个 Prompt 中
- ❌ 跨模块的需求

## 模块开发 Prompt 模板

以下模板供所有模块开发直接复制使用：

```markdown
# web-xiaobaigame vX.Y.Z
# [模块名称] 开发

========================
开发目标
========================

[描述本次开发目标]

========================
首先执行
========================

请先阅读以下文档：

1. AGENTS.md
2. README.md
3. docs/PROJECT.md
4. docs/MODULES.md
5. docs/ARCHITECTURE.md
6. docs/DEVELOPMENT_STATUS.md
7. docs/AI_MEMORY.md
8. [其他相关文档]

阅读完成后，输出【项目理解】。

========================
开发任务
========================

### 任务一：[任务名称]
[详细描述]

### 任务二：[任务名称]
[详细描述]

========================
开发纪律
========================

不得修改其他模块。
不得修改已完成模块。
不得修改历史 Migration。
不得删除已有文件。
不得提前开发下一模块。

必须同步更新：
- CHANGELOG.md
- MODULES.md
- DEVELOPMENT_STATUS.md
- [其他需要更新的文档]

========================
完成标准
========================

[描述完成的验收标准]
```

## 文档更新 Prompt 模板

```markdown
# web-xiaobaigame vX.Y.Z
# 文档优化

========================
开发目标
========================

本次仅完善项目文档。

禁止开发任何业务功能。
禁止修改任何业务代码。
禁止新增数据库/API/页面/组件。

========================
任务列表
========================

### 任务一：[文档名] 优化
[详细描述]

========================
开发纪律
========================

不得开发业务。
不得新增数据库/API/页面/组件。
必须同步更新 README.md 文档索引。

========================
完成标准
========================

完成后输出变更清单。
```

## Prompt 检查清单

提交 Prompt 前，确认以下项目：

- [ ] 是否包含「首先执行：阅读文档」步骤
- [ ] 是否明确了开发目标
- [ ] 是否列出了禁止事项
- [ ] 是否明确了需要更新的文档
- [ ] 是否明确了完成标准
- [ ] 是否单一模块/单一目标
- [ ] 是否避免了模糊表述
