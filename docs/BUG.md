## Bug 报告格式

```markdown
### BUG-XXX: 简短描述

- **发现日期**：YYYY-MM-DD
- **报告人**：@username
- **严重程度**：Critical / High / Medium / Low
- **影响范围**：哪个模块/功能
- **复现步骤**：
  1. ...
  2. ...
- **预期行为**：
- **实际行为**：
- **临时方案**：
- **修复版本**：vX.Y.Z
- **状态**：Open / In Progress / Fixed / Won't Fix
```

---

## 严重程度定义

| 级别 | 定义 |
|------|------|
| Critical | 系统崩溃、数据丢失、安全漏洞 |
| High | 核心功能不可用 |
| Medium | 功能部分不可用，有替代方案 |
| Low | 体验问题、不影响功能 |
